# 项目需求 - BugTrack Platform (MVP)

## 一、目标
构建一个简易的缺陷管理（Bug Tracking）与团队协作平台，实现缺陷的完整生命周期管理与基础项目协作功能。面向小团队、轻量级、易扩展。

## 二、MVP 范围（必做）
1. 项目管理
   - 创建 / 编辑 / 归档项目
   - 项目成员管理（添加/删除/角色）
2. 用户系统
   - 注册 / 登录 / 登出（基础身份验证）
   - 用户信息管理（昵称、邮箱）
3. 缺陷管理（Issues）
   - 新建缺陷（标题、描述、严重程度、优先级、模块、附件）
   - 缺陷状态流转（new → in_progress → resolved → verified → closed / reopened）
   - 指派（assignee）、报告人（reporter）
   - 搜索与筛选（按状态、指派人、优先级、创建时间等）
   - 缺陷编号（project + issue_no 显示）
4. 评论与协作
   - 缺陷下的评论（支持 @ 用户）
   - 评论显示时间线
5. 附件上传（少量文件，先实现本地/MinIO 存储接口）
6. 基础通知（站内通知：被指派 / 被@ / 状态变更）
7. 操作记录（issue_history：记录关键变更）
8. 权限控制（基于项目成员角色的基本权限）
9. RESTful API（后端接口，为前端提供数据）
10. 部署与运行说明（Docker 或直接 jar 部署）

## 三、非 MVP（可选/后续）
- 邮件通知 / 第三方集成（钉钉、飞书）
- 报表与统计（趋势图、成员绩效）
- 实时推送（WebSocket）
- 高级权限与组织结构
- 多语言支持
- CI/CD 集成

## 四、关键用例（User Stories）
- 作为项目成员，我可以创建并编辑缺陷。
- 作为报告人，我可以上传缺陷复现步骤和截图。
- 作为指派人，我会收到任务分配通知。
- 作为项目管理员，我可以管理项目成员与角色。
- 作为普通成员，我只能对我有权限的项目进行操作。

## 五、验收标准
- 能够在 1 个部署实例上，完成用户注册、创建项目、创建缺陷、指派与评论等完整流程。
- API 文档完整（Postman/Swagger）。
- 最低权限控制可生效（仅项目成员能访问项目内资源）。
